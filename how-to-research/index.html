1. 最初に知っておくべきこと

1.1 研究とは何か

まず、「研究」を定義するのに必要な概念をいくつか導入します。
以下の図のような状況を考えてください。

図1のような状況に置かれたとき、我々は目の前のモノに対して自然と興味を持ちます。
やがて、以下の図のように、いくつかの性質を発見します。

図2に示したような性質は、少しの労力で容易に明らかにできるものです。
このような少しの労力で容易に明らかにできる性質のことを自明な性質と呼びます。

しかしモノが持つのは自明な性質だけではありません。
以下の図のような状況を考えてみましょう。

図3の [？] の部分を埋めるには、世界の全地点の植生を調査する必要があります。
これは容易にできることではありません。
このように、モノが持つ性質の中には、自明な性質と違って容易に明らかにできないものがあります。
このような容易に明らかにできない性質のことを非自明な性質と呼びます。

以上を踏まえて、「研究」を定義します。

研究とは、着目している対象が持つ非自明な性質を明らかにすることです。
全ての分野の研究は上記の定義で理解できます。
図3における「植物」を
「P2Pアルゴリズム」などに置き換えると情報系の研究になります。
「素粒子」などに置き換えると物理系の研究になります。

新しい何かを作ることや、新しい何かを使うことは、
研究の一工程にすぎず、研究そのものではありません。
何らかの非自明な性質が明らかになっていなければ、研究ではありません。

ここまでは絶対に押さえてください。

1.2 非自明な性質をどう明らかにするか

非自明な性質を明らかにする方法には、
「疑問を設定して、それを解決する」「偶然発見する」の2通りがあります。

1.2.1 疑問を設定して、それを解決する

「この植物は世界のどこに分布している？」というような、
着目している対象が持つ非自明な性質に関する疑問をリサーチクエスチョンと呼びます。
リサーチクエスチョンを設定し、それを解決することで非自明な性質を明らかにするのが研究の基本的な戦略となります。

1.2.2 偶然発見する

これはリサーチクエスチョンを解決しようとしている時に偶然起きるものです。
狙うものではありませんが、結構よく起きます。

偶然による非自明な性質の発見の例としては、
導電性ポリアセチレンの発見やグラフェンの発見があります。

1.3 「非自明な性質であれば何でも良い」←×

明らかにしようとしているのが非自明な性質であれば何でも良いかというと、そうではありません。

この世には自分以外にもたくさんの人間がいます。
今自分が明らかにしようとしている非自明な性質は、既に他人によって明らかにされているかもしれません。

過去に他人が明らかにした非自明な性質を再び明らかにするというのは、
本人にとっては良い研究の訓練になりますが、人類全体にとっては無駄な行為です。
なぜなら、人類全体の知識に何の変化ももたらさないからです。

過去に他人が明らかにした非自明な性質を再び明らかにしてしまった場合、その研究は新規性がないと言います。
逆に、まだ誰にも明らかにされていない非自明な性質を明らかにした場合、その研究は新規性があると言います。

大学のような学術機関では、新規性のある研究を行うことが求められます。
なぜなら、新規性のある研究を行うことで人類全体の知識に変化をもたらすことが学術機関の仕事だからです。

1.4 「新規性があれば何でも良い」←×

新規性があれば何でも良いかというと、そうではありません。

例えば、Web ブラウザに着目し、Scala で Web ブラウザを作るとどうなるかに興味を持ったとしましょう。
Scala で Web ブラウザを作るとどうなるかは非自明です。そして恐らくまだ誰にも知られていません。
よってそれを明らかにすることは新規性のある研究となります。

仮にその研究によって、
「Scala で Web ブラウザを作ろうとすると、○○機能の実装が困難である」
というような性質が明らかになったとしましょう。

しかしそのような情報は、誰の役に立つでしょうか。
恐らく、Web ブラウザ研究者の役には立たず、
それこそ Scala で Web ブラウザを作ろうとしている人ぐらいにしか役に立たないでしょう。

このような、新規性はあるが、明らかにした性質がごく限られた人にしか役に立たない研究のことを、有用性の低い研究と言います。
逆に、新規性がある上に、明らかにした性質が数多くの人の役に立つ研究のことを、有用性の高い研究と言います。
（有用性の定義はたくさんあります。これが唯一の定義ではありません。）

有用性の高い研究の例としては、高速なソートアルゴリズムの発見が挙げられます。
「高速なソートアルゴリズムが存在する。また、その内容はこうである」という情報は、
アルゴリズム研究者にはもちろん、ソート処理を実装する全ての人に役立ちます。

大学のような学術機関では、有用性の高い研究を行うことが求められます。
なぜなら、学術機関は、単に人類全体の知識に変化をもたらすだけでなく、
有用性の高い研究を行うことで多くの研究者・非研究者に新たな知見を与えることが望ましいからです。

研究が持つ階層構造を1枚の図にまとめると、
以下の図のようになります。

大学の人間が「研究」という言葉を口にした場合、それは「新規性と高い有用性を兼ね備えた研究」のことを指します。

2. 何から手を付ければ良いのか

「非自明で」「まだ誰にも明らかにされていなくて」「有用な」性質を明らかにすれば良いのだ、
ということが分かっていても、
いざ始めてみると最初は「何から手を付ければ良いのか分からない」状態に陥ります。

しかし、そのような状態からは速攻で脱出するべきです。
さっさと脱出して、研究そのものに時間を費やすべきです。

そこで、この章では、「何から手を付ければ良いのか分からない」状態から速攻で脱出するための情報として、
何から手を付ければ良いのかを具体的に説明します。

2.1 着目する対象を具体的にする

何に着目するかが漠然としている場合は、まずそこをなんとかする必要があります。
なぜなら、研究は「着目している対象が持つ非自明な性質を明らかにすること」なので、
着目する対象を具体的に決めておかないと、研究を始められないからです。
そのため以下の図のように、着目する対象を具体化する必要があります。

なお、着目する対象には元と集合の2種類があることに注意してください。
「Scala」や「Twitter ユーザの F/F 関係グラフ」に着目した場合は、元に着目していることになります。
「P2Pアルゴリズム」や「複雑ネットワーク」に着目した場合は、集合に着目していることになります。

着目しようとしている対象が元なのか集合なのかは、
「ある○○」の「○○」に、対象を表す言葉を入れれば分かります。
違和感が生じれば元です。
違和感が生じなければ集合です。
例えば、「あるScala」という表現には違和感が生じますが、
「あるP2Pアルゴリズム」という表現には違和感は生じません。
ここから、「Scala」は元で、「P2Pアルゴリズム」は集合であることが分かります。

有用性の高い研究は、「考えうる全てのソートアルゴリズムの集合」などといった、（巨大な）集合に着目しているケースがほとんどです。
なぜなら、元に着目しても、特定の元の非自明な性質が明らかになるだけで、多くの人の役に立つ情報は得られないからです。
有用性の高い研究を行いたいならば、元ではなく集合に着目することをお勧めします。
（もちろん例外はあります。リーマン予想における命題は元ですが、その非自明な性質に関する情報は非常に有用です）

2.1.1 着目する対象の候補を集める

着目する対象を具体化しようにも、最初は着目する対象の候補すらありません。
そのため具体化の際は、まず候補を集めることになります。

候補を集める際は、過去に研究者が着目してきた対象を知るのが得策です。
自分の頭の中から候補をひねり出すよりも遥かに短時間で大量の候補を得ることができるからです。

では、過去に研究者が着目してきた対象を知るにはどうすれば良いでしょうか？
ここで初めて論文を読むタイミングがやってきます。
論文というのは自分が行った・行っている研究を第三者が再体験できるように、研究の要点を記したものです。
すると当然、論文にはその研究において著者が着目した対象も記されていることになります。
となると、自分が興味を持っている分野の論文から「その研究において著者が着目した対象」をひたすら抽出すれば、
過去に研究者が着目してきた対象を一網打尽に把握できることになります。
（以上の記述では「自分が興味を持っている分野の論文をまずどう探せばいいんだ」という話になりますが、そこら辺は～の章に回します）

以上の考え方により、着目する対象の候補を集めることができます。

しかし、この工程はそれなりに時間がかかる上に、
英語力や専門知識が不足していると途中で心が折れます。
学部生は指導教員や先輩から候補を貰って、この工程をスキップしてください。
修士学生も可能ならばスキップすることをお勧めします。

2.1.2 着目する対象を選択する

着目する対象の候補が手元に揃ったら、その中からどれか1つを選択しましょう。
一度着目した対象とは少なくとも数ヶ月以上付き合うことになるわけなので、
選択にはとても悩むはずですが、あまり時間をかけすぎないようにしてください。

2.2 着目した対象を深く理解する

何に着目するかが具体的に決まって、次にやるべきことは、着目した対象を深く理解することです。
なぜなら、研究は「着目している対象が持つ非自明な性質を明らかにすること」なので、
少なくとも自明な性質は知っておく必要があるからです。
さらに、新規性のある研究を行いたいならば、
自明な性質に加えて、過去に他人が明らかにした非自明な性質についてもある程度知っておく必要があります。

2.2.3 図を描いて直観的に捉える（着目した対象が無形物の場合）

情報系の研究において着目するのはたいてい無形物（⇔ 有形物）です。
無形物の例としては、アルゴリズム、ソフトウェア、行為、命題、関数、関係などがあります。

2.2.5 ルーツと歴史を調べる

人間の作り出した・考え出したモノには必ずルーツと歴史があります。
着目した対象を深く理解する際には、その対象のルーツと歴史を調べるのが有効です。
なぜなら、ルーツと歴史を調べることで、
その対象に対して過去に行われた研究を時系列順に再体験することができるからです。

この「時系列順に」という部分が重要です。
人間の作り出した・考え出したモノは、発明されたが最後、時を経るごとに複雑かつ大規模になっていきます（CPUがその典型です）。
そのため、着目した対象に対して現在行われている複雑怪奇な研究をいきなり再体験しようとするのはもはや無謀です。
一方、着目した対象に対して初期の初期に行われていた研究は、現在のものと比べて非常に単純な場合が多いです。
そのため、初期の初期の研究（単純な研究）から現在の研究（複雑怪奇な研究）までを時系列順に追うことで、
途中でつまずくことなく過去の研究を再体験することが可能になります。
「時系列順に」という部分が重要なのはそのためです。

ルーツと歴史の調査は、英語版Wikipediaの記事を起点に行うと楽です。
英語版Wikipediaの記事は日本語版Wikipediaの記事よりも「History」の章が圧倒的に充実しています。
（もちろん Wikipedia は完全に信頼できる情報源ではありませんので、出典の怪しい記述は適宜他の情報源を使って検証してください）

2.2.6 他研究者による説明を参考にする

論文にはたいてい、「その研究において著者が着目した対象」が持つ自明・非自明な性質の説明が添えられています。
それを参考にすれば、自力でイチから調べて考えるよりも、高速に理解を深めることができます。

2.3 リサーチクエスチョンを用意する

着目した対象を深く理解して、次にやるべきことは、リサーチクエスチョンを用意することです。
前章でも説明したように、
リサーチクエスチョンを設定し、それを解決することで非自明な性質を明らかにするのが研究の基本的な戦略となるからです。

リサーチクエスチョンを用意する際も、過去に研究者が設定してきたリサーチクエスチョンをまず知るのが得策です。

論文には、その研究において著者が着目した対象だけでなく、その研究において著者が設定したリサーチクエスチョンも記されています。
となると、自分が興味を持っている分野の論文から「その研究において著者が設定したリサーチクエスチョン」をひたすら抽出すれば、
過去に研究者が設定してきたリサーチクエスチョンを一網打尽に把握できることになります。

もちろん、自分で新たにリサーチクエスチョンを発想するのもアリです。
新たなリサーチクエスチョンは、それを発想しただけで価値が生じます。
新たなリサーチクエスチョンは、多くの派生研究を生み出すからです。
しかし、新たなリサーチクエスチョンの発想には着目した対象に関する深遠な理解が必要なので、
挑戦する際は覚悟してください。
